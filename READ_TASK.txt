Приложение можно запускать следующими способами

1. непосредственно открыть файл index.html в браузере 

2. Используя пакет browser-sync

Для этого необходимо установить пакет
Открыть терминал (например gitbush) и набрать команду 

    npm install -g  browser-sync

---- Для MacOS, Linux
    sudo npm install -g  browser-sync

Запуск приложения
- открыть терминал в каталоге сontrol_work
- запустить проект коммандой 
	
	npm start


==========================  ЗАДАНИЕ СЛУШАТЕЛЯМ

В файле  js/script.js есть список задач -tasks
В файле index.html есть готовая верстка приложения в котором 
показано какой должен быть код для невыполненной и выполненной задачи 

!!! НЕ вносить изменения в index.html код 
Код пишется только в файле js/script.js 

Реализовать:
1.  Вывод списка задач в элемент ul с id = tasks-box
    Если статус задачи ВЫПОЛНЕНА (свойство done = true) то
    и ее название (в элементе span) отображается красным цветом (класс done)
 

2. Отображение в блоке c id=tasks-info в соответствующем элементе span 
  - общее количество задач (id="tasks-total")
  - количество выполненных задач  (id="tasks-done")
  - количество не выполненных задач (id="tasks-remain")

  Если задачи отсутствуют блок task-info не должен отображаться 

3. При клике на кнопку Add Task добавление задачи, имя которой введено 
    в поле Add task 
    Имя задачи не может быть менее 5 символов
    
4. При клике на иконку с классом fa-check задаче присваивается статус ВЫПОЛНЕНА
   ее название (в элементе span) отображается красным цветом (класс done)       

   При клике на иконку с классом fa-check для задачи с статусом ВЫПОЛНЕНА 
   свойство done = false и  и ее название отображается цветом по умолчанию

5. При клике на иконку с классом fa-trash задача удаляется из списка 
   Перед удалением запросить подтвержение от пользователя через
   модальное окно confirm   

6.  Фильтрацию задач и их отображение при изменении элемента select с id=filter-tasks

7.  Фильтрацию задач по title и их отображение при вводе текста
   в поле c id = search-box  (по событию keyup)
